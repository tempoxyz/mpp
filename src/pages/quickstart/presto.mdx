# presto [Make paid HTTP requests from the command line]

`presto` is a command-line tool for making HTTP requests with automatic payment support. It handles `402 Payment Required` responses with built-in payment methods. It can be used from scripts, cron jobs, and AI agents.

## Quickstart

::::steps

### Install

```bash [install.sh]
$ curl -fsSL https://presto-binaries.tempo.xyz/install.sh | bash
```

### Log in

Run the login command to set up your wallet:

```bash [login.sh]
$ presto login
```

This opens your browser to authenticate with your Tempo wallet. Credentials are stored in your platform's native data directory.

### Make your first paid request

```bash [paid-request.sh]
$ presto -vv https://openrouter.payments.tempo.xyz/v1/chat/completions -d '{"model":"gpt-4o-mini","max_tokens":50,"messages":[{"role":"user","content":"Hello!"}]}'
```

:::tip[Live demo server]
We host a live demo server at `mpp.tempo.xyz` running on **Tempo testnet**. Use `/api/ping` for free requests and `/api/ping/paid` for paid requests. Get testnet funds at [faucet.tempo.xyz](https://faucet.tempo.xyz).
:::

`presto` automatically:

1. Makes the initial request
2. Detects the `402 Payment Required` response
3. Parses the payment challenge
4. Signs a payment credential
5. Retries with the payment credential
6. Returns the response

::::

## Learn more

### Preview before paying

Use dry-run mode to see what would happen without actually paying:

```bash [dry-run.sh]
$ presto --dry-run https://mpp.tempo.xyz/api/ping/paid
```

### Common flags

| Flag | Description |
|------|-------------|
| `--dry-run` | Preview without paying |
| `-v` | Verbose output |
| `-i` | Include response headers |
| `-o <file>` | Save response to file |
| `-q` | Quiet mode (suppress logs) |
| `--output-format json` | Output in JSON format |
| `-X <METHOD>` | Custom HTTP method |
| `--json <DATA>` | Send JSON data (sets Content-Type automatically) |
| `-d <DATA>` | POST data (use `@filename` to read from file, `@-` for stdin) |
| `-H <HEADER>` | Add custom header (repeatable) |
| `--no-redirect` | Disable following redirects |
| `-m <SECONDS>` | Request timeout |

## Built-in skills

presto ships with built-in support for common AI agents. When used as a skill by AI assistants like Claude, presto enables agents to interact with paid APIs autonomously.

### How it works

AI agents can use `presto` to:

- **Access paid APIs**—Make requests to services that charge per-call
- **Query premium data**—Fetch information from paid data providers
- **Use commercial tools**—Interact with paid developer services

The agent calls `presto` like any other command-line tool when it detects a `402` Payment Required response. Payment is handled transparently.

### Example: Agent making a paid request

When an AI agent needs to access a paid resource, it runs:

```bash [agent-request.sh]
$ presto https://api.example.com/premium-data
```

The agent receives the response data directly—no manual payment handling required.

## Safety controls

presto includes safeguards for autonomous agent use:

### Dry run mode

Agents can preview costs before committing:

```bash [preview-cost.sh]
$ presto --dry-run https://api.example.com/expensive-endpoint
```

### Network filtering

Restrict payments to specific networks:

```bash
presto -n tempo-moderato https://api.example.com/data
```

### Skill configuration

For AI agent frameworks that support skills, presto can be configured as a tool:

```json
{
  "name": "presto",
  "description": "Make HTTP requests with automatic payment support",
  "parameters": {
    "url": {
      "type": "string",
      "description": "The URL to request"
    }
  }
}
```

## Use cases

### Premium API access

```bash [query-api.sh]
# Query a paid data API
$ presto https://api.data-provider.com/market-data?symbol=BTC
```

### Paid compute

```bash [inference.sh]
# Run inference on a paid model
$ presto -X POST \
  --json '{"prompt": "Explain quantum computing"}' \
  https://api.ai-provider.com/generate
```

### Commercial tools

```bash [analyze-repo.sh]
# Use a paid developer tool
$ presto https://api.dev-tools.com/analyze?repo=myorg/myrepo
```
