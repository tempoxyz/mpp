# Server quickstart [Charge for resources and verify payment credentials]

## Overview

Charge for resources using the `mpay` server SDK.

The server SDK provides two approaches:

| Approach | Use Case | Integration Shape |
|----------|----------|-------------------|
| [`Mpay.create`](/sdk/typescript/server/Mpay.create) | Creates a payment handler with configured method and realm | <Badge variant="info">Standard</Badge> |
| [`Mpay.toNodeListener`](/sdk/typescript/server/Mpay.toNodeListener) | Adapts the handler for Node.js HTTP servers | <Badge variant="note">Custom</Badge> |

## Basic setup

Create a payment handler with your payment method and server realm:

```ts [server.ts]
import { Mpay, tempo } from 'mpay/server'

const payment = Mpay.create({ // [!code hl]
  method: tempo({ // [!code hl]
    rpcUrl: 'https://rpc.tempo.xyz', // [!code hl]
    chainId: 42431, // [!code hl]
  }), // [!code hl]
  realm: 'mpp.dev', // [!code hl]
  secretKey: process.env.MPP_SECRET_KEY!, // [!code hl]
}) // [!code hl]
```

:::tip[Common configuration options]
- `method`: Payment method (for example, `tempo({ ... })`)
- `realm`: Server realm, typically your hostname
- `secretKey`: Secret for HMAC-bound challenge IDs (enables stateless verification)
:::

## Protecting a resource

Use intent methods to protect resources. When a request lacks valid payment credentials, the server returns a `402` challenge. Once payment is verified, the server returns the protected content with a receipt.

```ts [charge.ts]
// Define what to charge
const chargeHandler = payment.charge({
  request: {
    amount: '1000000',                                        // 1 USD (6 decimals)
    currency: '0x20c0000000000000000000000000000000000000',   // pathUSD
    recipient: '0x...',                                       // Your receiving address
  },
  description: 'API access',
})
```

### With a web framework

:::code-group

```ts [Bun]
import { Mpay, tempo } from 'mpay/server'

const payment = Mpay.create({
  method: tempo({ rpcUrl: 'https://rpc.tempo.xyz', chainId: 42431 }),
  realm: 'mpp.dev',
  secretKey: process.env.MPP_SECRET_KEY!,
})

export default {
  async fetch(request: Request): Promise<Response> {
    const result = await payment.charge({ // [!code hl]
      request: { // [!code hl]
        amount: '1000000', // [!code hl]
        currency: '0x20c0000000000000000000000000000000000000', // [!code hl]
        recipient: '0x...', // [!code hl]
      }, // [!code hl]
    })(request) // [!code hl]

    if (result.status === 402) return result.challenge // [!code hl]

    return result.withReceipt( // [!code hl]
      Response.json({ data: 'protected content' })
    )
  }
}
```

```ts [Node]
import * as http from 'node:http'
import { Mpay, tempo } from 'mpay/server'

const payment = Mpay.create({
  method: tempo({ rpcUrl: 'https://rpc.tempo.xyz', chainId: 42431 }),
  realm: 'mpp.dev',
  secretKey: process.env.MPP_SECRET_KEY!,
})

http.createServer(async (req, res) => {
  const result = await Mpay.toNodeListener( // [!code hl]
    payment.charge({ // [!code hl]
      request: { // [!code hl]
        amount: '1000000', // [!code hl]
        currency: '0x20c0000000000000000000000000000000000000', // [!code hl]
        recipient: '0x...', // [!code hl]
      }, // [!code hl]
    }), // [!code hl]
  )(req, res) // [!code hl]

  if (result.status === 402) return // [!code hl]

  res.setHeader('Content-Type', 'application/json')
  res.end(JSON.stringify({ data: 'protected content' }))
}).listen(3000)
```

## Node.js HTTP server

For Node.js HTTP servers, use `Mpay.toNodeListener` to adapt the payment handler:

```ts [node-server.ts]
import * as http from 'node:http'
import { Mpay, tempo } from 'mpay/server'

const payment = Mpay.create({
  method: tempo({
    rpcUrl: 'https://rpc.tempo.xyz',
    chainId: 42431,
  }),
  realm: 'mpp.dev',
  secretKey: process.env.MPP_SECRET_KEY!,
})

http.createServer(async (req, res) => {
  const result = await Mpay.toNodeListener( // [!code hl]
    payment.charge({ // [!code hl]
      request: { // [!code hl]
        amount: '1000000', // [!code hl]
        currency: '0x20c0000000000000000000000000000000000000', // [!code hl]
        recipient: '0x...', // [!code hl]
      }, // [!code hl]
    }), // [!code hl]
  )(req, res) // [!code hl]

  // 402 response already written
  if (result.status === 402) return

  // Write protected content
  res.setHeader('Content-Type', 'application/json')
  res.end(JSON.stringify({ data: 'protected content' }))
}).listen(3000)
```

:::warning
`Mpay.toNodeListener` writes the 402 response directly to the response object. Always check `result.status` before writing your own response.
:::

## Testing with pget

After your server is running, test it with [pget](/tools/pget):

```bash [test.sh]
# See the payment challenge
pget inspect <your-server>/resource

# Make a paid request (requires funded wallet)
pget <your-server>/resource

# Verbose output to see the full 402 flow
pget -vi <your-server>/resource

# Dry run to preview payment without executing
pget -D <your-server>/resource
```

:::tip
Use `pget inspect` to debug your server's challenge response without making any payments.
:::
