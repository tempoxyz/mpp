# Client quickstart [Handle payment-gated resources automatically]

Polyfill the global `fetch` to handle `402` responses. Your existing code works unchanged—payments happen in the background.

## Steps

::::steps

### Install dependencies

:::code-group
```bash [npm]
npm install mpay viem
```
```bash [pnpm]
pnpm add mpay viem
```
```bash [bun]
bun add mpay viem
```
:::

### Define an account

```ts twoslash [define-account.ts]
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0xabc…123')
```

:::tip
With Tempo, you can also use [Passkey or WebCrypto accounts](https://viem.sh/tempo/accounts).
:::

### Polyfill fetch

Call `Fetch.polyfill` at startup. All `fetch` calls then handle payment challenges.

```ts twoslash [polyfill-fetch.ts]
import { privateKeyToAccount } from 'viem/accounts'
import { Fetch, tempo } from 'mpay/client'

const account = privateKeyToAccount('0xabc…123')

Fetch.polyfill({
  methods: [tempo({ account })]
})
```

### Request protected resources

Use `fetch`. Payment happens when a server returns `402`.

```ts [fetch-resource.ts]
const response = await fetch('https://mpp.dev/ping/paid')
```

## Advanced options

### Wrapped fetch instance

Use `Fetch.from` to create a separate payment-enabled fetch without modifying the global:

```ts twoslash [wrapped-fetch.ts]
import { privateKeyToAccount } from 'viem/accounts'
import { Fetch, tempo } from 'mpay/client'

const account = privateKeyToAccount('0xabc…123')

// [!code hl:start]
const paidFetch = Fetch.from({
  methods: [tempo({ account })]
})
// [!code hl:end]
const response = await paidFetch('https://mpp.dev/ping/paid')
```

### Per-request accounts

Pass accounts on individual requests instead of at setup:

```ts twoslash [per-request-account.ts]
import { privateKeyToAccount } from 'viem/accounts'
import { Fetch, tempo } from 'mpay/client'

const paidFetch = Fetch.from({
  methods: [tempo()]
})

// [!code hl:start]
const response = await paidFetch('https://mpp.dev/ping/paid', {
  context: {
    account: privateKeyToAccount('0xabc…123'),
  }
  // [!code hl:end]
})
```

### Manual payment handling

Use `Mpay.create` for full control over the payment flow:

- Present payment UI before paying
- Implement custom retry logic
- Handle credentials manually

```ts twoslash [manual.ts]
import { Mpay, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

// [!code hl:start]
const mpay = Mpay.create({
  methods: [tempo()],
})
// [!code hl:end]

const response = await fetch('https://mpp.dev/ping/paid')

if (response.status === 402) {
  const credential = await mpay.createCredential(response, {
    account: privateKeyToAccount('0x...'),
  })

  const paidResponse = await fetch('https://mpp.dev/ping/paid', {
    headers: { Authorization: credential },
  })
}
```

### Payment receipts

On success, the server returns a `Payment-Receipt` header:

```ts [receipts.ts]
import { Receipt } from 'mpay'

const header = response.headers.get('Payment-Receipt') // [!code hl]
const receipt = Receipt.deserialize(header)
console.log(receipt.status)     // 'success'
console.log(receipt.reference)  // transaction hash
console.log(receipt.timestamp)  // ISO 8601
```
