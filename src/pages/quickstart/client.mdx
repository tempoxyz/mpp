# Client quickstart [Pay for resources using the MPP client SDK]

## Overview

Agents and other programmatic flows can pay for services using the `mpay` client SDK. The SDK provides programmatic tools for paying with MPP:

| Approach | Use Case | Integration Shape |
|----------|----------|-------------|
| [`Fetch.from`](/sdk/typescript/client/Fetch.from) | Wraps fetch to automatically handle `402` responses (recommended) | <Badge variant="info">Standard</Badge> |
| [`Mpay.create`](/sdk/typescript/client/Mpay.create) | Lower-level API for manual control over payment flow | <Badge variant="note">Custom</Badge> |

## Use the Fetch wrapper

To pay with MPP, use `mpay`'s fetch wrapper. When the server returns `402`, the wrapped fetch automatically creates a payment credential and retries.

```ts [client.ts]
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const fetch = Fetch.from({ // [!code hl]
  methods: [ // [!code hl]
    tempo({ // [!code hl]
      account: privateKeyToAccount('0x...'), // [!code hl]
      rpcUrl: 'https://rpc.tempo.xyz', // [!code hl]
    }), // [!code hl]
  ],// [!code hl]
}) // [!code hl]

// Handles 402 automatically—pays and retries
const response = await fetch('https://mpp.dev/ping/paid')
const data = await response.json()
```


## Global fetch polyfill


`mpay` ships with polyfill support for existing client applications.

Replace the global `fetch` with a payment-aware version:

```ts [polyfill.ts]
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

// [!code hl]
Fetch.polyfill({ // [!code hl]
  methods: [ // [!code hl]
    tempo({ // [!code hl]
      account: privateKeyToAccount('0x...'), // [!code hl]
      rpcUrl: 'https://rpc.tempo.xyz', // [!code hl]
    }), // [!code hl] 
  ], // [!code hl] 
}) // [!code hl] 

// Global fetch now handles 402 automatically
const response = await fetch('https://mpp.dev/ping/paid')

// Restore original fetch when done
Fetch.restore()
```

:::warning
The global polyfill affects all `fetch` calls in your application. Use `Fetch.from` for more targeted control in complex workflows.
:::

## Manual payment handling
For more control, use `Mpay.create` directly:

This is useful in the following scenarios:

- Present payment UI before paying
- Use different accounts per request
- Implement custom retry logic


```ts [manual.ts]
import { Mpay, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const mpay = Mpay.create({ // [!code hl]
  methods: [ // [!code hl]
    tempo({ // [!code hl]
      rpcUrl: 'https://rpc.tempo.xyz', // [!code hl]
    }), // [!code hl]
  ], // [!code hl]
}) // [!code hl]

// Make request
const response = await fetch('https://mpp.dev/ping/paid')

// Check if payment is required
if (response.status === 402) { // [!code focus:10]
  // Create credential from the challenge
  const credential = await mpay.createCredential(response, {
    account: privateKeyToAccount('0x...'),
  })

  // Retry with credential
  const paidResponse = await fetch('https://mpp.dev/ping/paid', {
    headers: { Authorization: credential },
  })
}
```

## Payment receipts

After successful payment, the server returns a `Payment-Receipt` header:

```ts [receipts.ts]
import { Receipt } from 'mpay'

const receiptHeader = response.headers.get('Payment-Receipt') // [!code hl]
if (receiptHeader) {
  const receipt = Receipt.deserialize(receiptHeader)
  console.log(receipt.status)     // 'success' | 'failed'
  console.log(receipt.reference)  // transaction hash
  console.log(receipt.timestamp)  // ISO 8601
}
```
