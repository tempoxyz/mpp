# Client quickstart [Handle payment-gated resources automatically]

## Overview

You can pay for services programmatically using the `mpay` client SDK. The SDK provides two approaches:

| Approach | Use Case | Integration Shape |
|----------|----------|-------------|
| [`Fetch.from`](#create-a-payment-enabled-fetch) | Wraps fetch to automatically handle `402` responses (recommended) | <Badge variant="info">Standard</Badge> |
| [`Mpay.create`](#manual-payment-handling) | Lower-level API for manual control over payment flow | <Badge variant="note">Custom</Badge> |

## Steps

::::steps

### Install dependencies

Install `mpay` and `viem`.

:::code-group
```bash [npm]
npm install mpay viem
```
```bash [pnpm]
pnpm add mpay viem
```
```bash [bun]
bun add mpay viem
```
:::

### Define an account

Next, define an account that will be used to sign payments.

```ts twoslash [example.ts]
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0xabc…123')
```

:::tip
When using the Tempo payment method, you can also use [Passkey or WebCrypto accounts](https://viem.sh/tempo/accounts).
:::

### Create a payment-enabled `fetch`

Create a payment-enabled `fetch` instance to handle requests that require payment. 

Pass your Tempo `account` to the `fetch` instance and specify the payment `method`(s) you want to use.

```ts twoslash [example.ts]
import { privateKeyToAccount } from 'viem/accounts'
// [!code focus:start]
import { Fetch, tempo } from 'mpay/client'
// [!code focus:end]

const account = privateKeyToAccount('0xabc…123')

// [!code focus:start]
const fetch = Fetch.from({
  methods: [
    tempo({ account })
  ]
})
// [!code focus:end]
```

:::tip
You can also polyfill the global `fetch` instance.

```ts 
Fetch.polyfill({
  methods: [
    tempo({ account })
  ]
})

// Now, the global `fetch` instance is payment-enabled.
const response = await fetch('https://mpp.dev/ping/paid')
```
:::

### Request protected resources

Use your `fetch` instance to make requests to resources that require payment.

```ts [example.ts]
const response = await fetch('https://mpp.dev/ping/paid')
```

:::tip
You can also define accounts on a per-request basis under `context`.

```ts 
const fetch = Fetch.from({ 
  account, // [!code --] 
  methods: [ 
    tempo({ account }) 
  ] 
}) 

const response = await fetch('https://mpp.dev/ping/paid', {
  context: { 
    account, // [!code ++] 
  } 
})
```
:::

## Learn more

### Manual payment handling

For more control, use `Mpay.create(){:ts}` directly:

This is useful in the following scenarios:

- Present payment UI before paying
- Use different accounts per request
- Implement custom retry logic


```ts twoslash [manual.ts]
import { Mpay, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

// [!code focus:start]
const mpay = Mpay.create({ 
  methods: [ 
    tempo({ account: privateKeyToAccount('0x...') }), 
  ], 
}) 
// [!code focus:end]

// Make request
const response = await fetch('https://mpp.dev/ping/paid')

// Check if payment is required
if (response.status === 402) { 
  // [!code focus:start]
  // Create credential from the challenge
  const credential = await mpay.createCredential(response, {
    account: privateKeyToAccount('0x...'),
  })
  // [!code focus:end]

  // Retry with credential
  const paidResponse = await fetch('https://mpp.dev/ping/paid', {
    headers: { Authorization: credential },
  })
}
```

### Payment receipts

After successful payment, the server returns a `Payment-Receipt` header:

```ts [receipts.ts]
import { Receipt } from 'mpay'

const header = response.headers.get('Payment-Receipt') // [!code hl]
const receipt = Receipt.deserialize(header)
console.log(receipt.status)     // 'success' | 'failed'
console.log(receipt.reference)  // transaction hash
console.log(receipt.timestamp)  // ISO 8601
```
