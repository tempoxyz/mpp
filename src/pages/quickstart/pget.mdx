# pget [wget for paid APIs]

`pget` is a non-interactive commandline tool for making HTTP requests with automatic payment support. Handles `402 Payment Required` responses with built-in payment methods and permissions. Works from scripts, cron jobs, and AI agents.

## Quickstart

::::steps

### Install

```bash [Terminal]
$ curl -fsSL https://pget.tempo.xyz/install.sh | bash
```

### Initialize

Run the init wizard to set up your wallet:

```bash [Terminal]
$ pget init
```

This creates a new encrypted keystore and saves your configuration to `~/.pget/config.toml`.

:::tip[Quick setup for testing]
For quick testing without a keystore, set your private key directly:

```bash [Terminal]
$ export PGET_PRIVATE_KEY="0x..."
```

:::

### Make your first paid request

```bash [Terminal]
$ pget -vv https://mpp.dev/ping/paid
```

`pget` automatically:

1. Makes the initial request
2. Detects the `402` Payment Required response
3. Parses the payment challenge from `WWW-Authenticate`
4. Signs and submits the payment
5. Retries with the payment credential
6. Returns the response

::::

## Learn more

### Preview before paying

Use dry-run mode to see what would happen without actually paying:

```bash [Terminal]
$ pget -D https://mpp.dev/ping/paid
```

Or inspect the payment requirements before paying:

```bash [Terminal]
$ pget inspect https://mpp.dev/ping/paid
```

### Set spending limits

Protect yourself with a maximum payment amount:

```bash [Terminal]
# Reject payments over 1 USD
$ pget -M 1000000 https://mpp.dev/ping/paid
```

### Common flags

| Flag | Description |
|------|-------------|
| `-D` | Dry run - preview without paying |
| `-y` | Require confirmation before payment |
| `-M <amount>` | Maximum payment amount (atomic units) |
| `-v` | Verbose output |
| `-i` | Include response headers |
| `-o <file>` | Save response to file |
| `-n <network>` | Filter to specific network |

## Built-in skills

pget ships with built-in support for common AI agents. When used as a skill by AI assistants like Claude, pget enables agents to interact with paid APIs autonomously.

### How it works

AI agents can use `pget` to:

- **Access paid APIs** — Make requests to services that charge per-call
- **Query premium data** — Fetch information from paid data providers
- **Use commercial tools** — Interact with paid developer services

The agent calls pget like any other commandline tool when it detects a `402` Payment Required response. Payment is handled transparently.

### Example: Agent making a paid request

When an AI agent needs to access a paid resource, it runs:

```bash [Terminal]
$ pget https://api.example.com/premium-data
```

The agent receives the response data directly—no manual payment handling required.

## Safety controls

pget includes safeguards for autonomous agent use:

### Spending limits

Set a maximum amount the agent can spend per request:

```bash [Terminal]
# Limit to 0.50 USDC per request
$ pget -M 500000 https://api.example.com/data
```

### Dry run mode

Agents can preview costs before committing:

```bash [Terminal]
$ pget -D https://api.example.com/expensive-endpoint
```

### Network filtering

Restrict payments to specific networks:

```bash
pget -n tempo-moderato https://api.example.com/data
```

### Skill configuration

For AI agent frameworks that support skills, pget can be configured as a tool:

```json
{
  "name": "pget",
  "description": "Make HTTP requests with automatic payment support",
  "parameters": {
    "url": {
      "type": "string",
      "description": "The URL to request"
    },
    "max_payment": {
      "type": "integer",
      "description": "Maximum payment in atomic units (optional)"
    }
  }
}
```

## Use cases

### Premium API access

```bash
# Query a paid data API
pget https://api.data-provider.com/market-data?symbol=BTC
```

### Paid compute

```bash
# Run inference on a paid model
pget -X POST \
  --json '{"prompt": "Explain quantum computing"}' \
  https://api.ai-provider.com/generate
```

### Commercial tools

```bash
# Use a paid developer tool
pget https://api.dev-tools.com/analyze?repo=myorg/myrepo
```
