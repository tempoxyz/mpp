# presto [Command-line HTTP client for MPP]

`presto` is a command-line HTTP client with automatic payment. On `402 Payment Required`, it handles payment and retries. No API keys needed.

## Install

```bash [install.sh]
$ curl -fsSL https://presto-binaries.tempo.xyz/install.sh | bash
```

## Setup

```bash [login.sh]
$ presto login
```

This opens your browser to authenticate with your Tempo wallet. Verify readiness:

```bash [preflight.sh]
$ presto whoami
```

Check `ready` is `true` and `key.balance` is sufficient. If not, run `presto login`.

## Available services

```bash [services.sh]
$ presto services
```

Browse and search the service directory:

```bash [services-search.sh]
# Filter by category
$ presto services --category ai

# Search by name or description
$ presto services --search "image generation"

# Show detailed info for a service
$ presto services info openai
```

Pattern: `<service>.mpp.tempo.xyz`. Examples:
- `https://openrouter.mpp.tempo.xyz/v1/chat/completions`
- `https://openai.mpp.tempo.xyz/v1/chat/completions`
- `https://anthropic.mpp.tempo.xyz/v1/messages`
- `https://fal.mpp.tempo.xyz/fal-ai/flux/schnell`

## Examples

```bash [examples.sh]
# LLM request
$ presto -X POST \
  --json '{"model":"openai/gpt-4o-mini","messages":[{"role":"user","content":"Hello"}]}' \
  https://openrouter.mpp.tempo.xyz/v1/chat/completions

# Image generation
$ presto -X POST \
  --json '{"prompt":"A sunset over mountains","image_size":"landscape_4_3","num_images":1}' \
  https://fal.mpp.tempo.xyz/fal-ai/flux/schnell

# Preview cost without paying
$ presto --dry-run -X POST \
  --json '{"model":"openai/gpt-4o-mini","messages":[{"role":"user","content":"Hello"}]}' \
  https://openrouter.mpp.tempo.xyz/v1/chat/completions
```

## Error recovery

| Exit code | Meaning | Action |
|-----------|---------|--------|
| 0 | Success | â€” |
| 1 | General error | Retry |
| 3 | Config error | `presto login` |
| 4 | Network error | Retry |
| 5 | Payment failed | Retry |
| 6 | Insufficient funds | Tell user to fund wallet |
| 8 | Auth error | `presto login` |
| 10 | Timeout | Retry with `-m <secs>` |

On `No wallet configured`, `Access key is not provisioned`, or exit 3/8: automatically run `presto login` then retry.
On `Spending limit exceeded` or `Insufficient balance`: report to user.

## Commands

| Command | Description |
|---------|-------------|
| `presto <url>` | HTTP request with automatic payment |
| `presto login` | Log in (opens browser) |
| `presto logout [--yes]` | Log out |
| `presto whoami` | Wallet status, balances, keys |
| `presto services` | Browse the MPP service directory |
| `presto services info <ID>` | Show detailed info for a service |
| `presto sessions list` | List sessions (active/orphaned/closing) |
| `presto sessions info <URL\|channel_id>` | Show details for a session or channel |
| `presto sessions close [--all\|--orphaned\|<URL>]` | Close sessions or channels |
| `presto sessions recover <URL>` | Re-sync a local session from chain |
| `presto sessions sync` | Remove stale local sessions |
| `presto update` | Update presto to the latest version |

### Display options

| Flag | Description |
|------|-------------|
| `-s`, `--silent` | Suppress non-essential output (recommended for agents) |
| `-j`, `--json-output` | JSON output |
| `-t`, `--toon-output` | TOON output (compact, token-efficientâ€”recommended for agents) |
| `-i` | Include response headers |
| `-o <FILE>` | Write output to file |
| `-v` | Verbose (`-vv` debug, `-vvv` trace) |

### HTTP options

| Flag | Description |
|------|-------------|
| `-X <METHOD>` | HTTP method |
| `--json <DATA>` | JSON body (sets Content-Type) |
| `--toon <TOON>` | TOON body (decoded to JSON, sets Content-Type) |
| `-d <DATA>` | POST data (`@file` or `@-` for stdin) |
| `--data-urlencode <DATA>` | URL-encode a data field (repeatable) |
| `-H <HEADER>` | Custom header (repeatable) |
| `-L` | Follow redirects |
| `-m <SECONDS>` | Timeout |
| `--connect-timeout <SECONDS>` | TCP connection timeout |
| `--retries <N>` | Retry on transient network errors |
| `--retry-backoff <MILLIS>` | Initial retry backoff (doubles each retry) |
| `--stream` | Stream response body as it arrives |
| `--sse` | Treat response as Server-Sent Events |
| `--sse-json` | Output each SSE event as NDJSON |
| `-f` | Fail on HTTP errors |
| `--compressed` | Request a compressed response |

### Payment options

| Flag | Description |
|------|-------------|
| `--dry-run` | Preview payment without executing |
| `--max-pay <AMOUNT>` | Cap the maximum amount to pay |
| `--currency <ADDR\|SYMBOL>` | Currency for `--max-pay` |
| `--save-receipt <FILE>` | Save payment receipt to file (JSON) |
