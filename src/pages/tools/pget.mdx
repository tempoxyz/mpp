# pget [wget for paid APIs]

`pget` is a non-interactive commandline tool for making HTTP requests with automatic payment support.

When you request a resource that requires payment, `pget` automatically detects the `402` response, fulfills the payment, and retries—all in a single command.

## Installation

**Quick install:**

```bash [install.sh]
$ curl -fsSL https://pget.tempo.xyz/install.sh | bash
```

**From source:**

```bash [build-from-source.sh]
$ git clone https://github.com/tempoxyz/pget.git
$ cd pget
$ cargo install --path .
```

## Quick start

```bash [quickstart.sh]
# Initialize (first time only)
$ pget init

# Make a paid request
$ pget https://mpp.tempo.xyz/ping/paid
```

:::tip[Live demo server]
We host a live demo server at `mpp.tempo.xyz` running on **Tempo testnet**. Use `/ping` for free requests and `/ping/paid` for paid requests. Get testnet funds at [faucet.tempo.xyz](https://faucet.tempo.xyz).
:::

## Configuration

`pget` uses encrypted keystores for secure wallet management.

### Example config

```toml
[evm]
keystore = "/Users/you/.pget/keystores/my-wallet.json"

# Override RPC URLs
[rpc]
tempo = "https://my-private-rpc.com"
```

## Payment method management

```bash [manage-methods.sh]
# List all payment methods
$ pget method list

# Create a new wallet
$ pget method new my-wallet --generate

# Import an existing private key
$ pget method import my-wallet

# Check balance on specific network
$ pget balance -n tempo
```

## Command reference

| Command | Alias | Description |
|---------|-------|-------------|
| `pget init` | `pget i` | Initialize configuration |
| `pget config` | `pget c` | View/manage configuration |
| `pget method` | `pget m` | Manage payment methods |
| `pget balance` | `pget b` | Check wallet balance |
| `pget networks` | `pget n` | List supported networks |
| `pget inspect` | — | Inspect payment requirements |
