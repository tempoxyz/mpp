# Installation 

`mpay` is the canonical SDK for the Machine Payments Protocol.

Install `mpay` via your package manager, a script tag, or build from source

## Package Manager

Install the required packages.

:::code-group

```bash [pnpm]
pnpm add mpay viem
```

```bash [npm]
npm install mpay viem
```

```bash [yarn]
yarn add mpay viem
```

```bash [bun]
bun add mpay viem
```

:::

## CDN

If you're not using a package manager, you can also use mpay via an ESM-compatible CDN such as esm.sh. Simply add a `<script type="module">` tag to the bottom of your HTML file with the following content.

```html
<script type="module">
  import { Challenge, Credential, Receipt } from 'https://esm.sh/mpay'
</script>
```

## Using Unreleased Commits

To test unreleased features, install from the canary tag (tracks the main branch).

:::code-group

```bash [pnpm]
pnpm add mpay@canary
```

```bash [npm]
npm install mpay@canary
```

```bash [yarn]
yarn add mpay@canary
```

```bash [bun]
bun add mpay@canary
```

:::

Or clone the mpay repo to your local machine, build, and link it yourself.

```bash
gh repo clone wevm/mpay
cd mpay
pnpm install
pnpm build
pnpm link --global
```

Then go to the project where you are using mpay and run `pnpm link --global mpay` (or the package manager that you used to link mpay globally).

## Security

Payment-related projects are often targeted in attacks to steal users' assets. Make sure you follow security best-practices for your project. Some quick things to get started.

- Pin package versions, upgrade mindfully, and inspect lockfile changes to minimize the risk of supply-chain attacks.
- Add a [Content Security Policy](https://developer.mozilla.org/en-US/docs/Web/HTTP/CSP) to defend against external scripts running in your app.
