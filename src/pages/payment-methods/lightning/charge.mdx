# Lightning charge [One-time payments using BOLT11 invoices]

The Lightning implementation of the [charge](/intents/charge) intent.

The server generates a fresh BOLT11 invoice for each request. The client pays it over the Lightning Network via a Spark wallet and presents the payment preimage as a credential. The server verifies `sha256(preimage) == paymentHash` locally ‚Äî no external service needed ‚Äî and returns the resource with a receipt.

This method is best for single API calls, content access, or one-off purchases.

## Server

Use `spark.charge` to gate any endpoint behind a one-time Lightning payment. The method handles invoice generation, Challenge creation, preimage verification, and Receipt generation.

```ts
import { Mppx, spark } from 'spark-mppx/server'

const mppx = Mppx.create({
  methods: [spark.charge({ mnemonic: process.env.MNEMONIC! })],
  secretKey: process.env.MPP_SECRET_KEY!,
})

export async function handler(request: Request) {
  const result = await mppx.charge({
    amount: '100',
    currency: 'BTC',
    description: 'Premium API access',
  })(request)

  if (result.status === 402) return result.challenge

  return result.withReceipt(Response.json({ data: '...' }))
}
```

### With expiry

```ts
import { Mppx, Expires, spark } from 'spark-mppx/server'

const mppx = Mppx.create({
  methods: [spark.charge({ mnemonic: process.env.MNEMONIC! })],
  secretKey: process.env.MPP_SECRET_KEY!,
})

export async function handler(request: Request) {
  const result = await mppx.charge({
    amount: '100',
    currency: 'BTC',
    expires: Expires.minutes(10), // [!code hl]
  })(request)

  if (result.status === 402) return result.challenge
  return result.withReceipt(Response.json({ data: '...' }))
}
```

## Client

Use `spark.charge` with `Mppx.create` to automatically handle `402` responses. The client parses the Challenge, pays the BOLT11 invoice via its Spark wallet, and retries with the preimage as a Credential.

```ts
import { Mppx, spark } from 'spark-mppx/client'

Mppx.create({
  methods: [spark.charge({ mnemonic: process.env.MNEMONIC! })],
})

const response = await fetch('https://api.example.com/resource')
```

### Without polyfill

If you don't want to patch `globalThis.fetch`, use `mppx.fetch` directly:

```ts
import { Mppx, spark } from 'spark-mppx/client'

const method = spark.charge({ mnemonic: process.env.MNEMONIC! })

const mppx = Mppx.create({
  polyfill: false,
  methods: [method],
})

try {
  const response = await mppx.fetch('https://api.example.com/resource')
  console.log(await response.json())
} finally {
  await method.cleanup()
}
```

:::info
The Spark SDK maintains WebSocket connections for Lightning payments. Call `method.cleanup()` when done to close connections and allow the process to exit.
:::

## Request fields

The Challenge request includes the base charge fields plus Lightning method details.

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `amount` | `string` | Yes | Invoice amount in satoshis |
| `currency` | `string` | No | Must be `'BTC'` if present. Defaults to `'BTC'` |
| `description` | `string` | No | Human-readable memo. Maps to the BOLT11 description field |
| `methodDetails.invoice` | `string` | Yes | Full BOLT11-encoded payment request (`lnbc...`). Authoritative source for all payment parameters |
| `methodDetails.paymentHash` | `string` | No | SHA-256 hash of the preimage, lowercase hex. Convenience field ‚Äî must match the hash decoded from `invoice` |
| `methodDetails.network` | `string` | No | `'mainnet'`, `'regtest'`, or `'signet'`. Convenience field ‚Äî must match `invoice`'s human-readable prefix. Defaults to `'mainnet'` |

## Credential payload

The Credential payload contains the payment preimage revealed by Lightning HTLC settlement.

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `preimage` | `string` | Yes | 32-byte payment preimage, lowercase hex (64 characters) |

## Verification

The server verifies payment with a single hash operation:

1. Decode the Credential and extract `preimage`.
2. Compute `sha256(hex_to_bytes(preimage))`.
3. Compare against the `paymentHash` from the original Challenge.
4. If equal, payment is verified. Return the resource with a Receipt.

No RPC calls. No external service. No database lookup in the verification path.
