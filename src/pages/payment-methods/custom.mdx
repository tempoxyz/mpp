# Custom [Build your own payment method]

The mpay SDK supports extensibility. You can implement custom payment methods to integrate any payment rails—Lightning Network, other blockchains, card processors, or proprietary systems.

## Overview

A custom payment method requires:

1. **Intent definition:** Schema for request parameters and Credential payloads
2. **Method Intent:** Extend a base Intent with method-specific schemas
3. **Client logic:** Create Credentials when the client gets a `402` response
4. **Server logic:** Verify Credentials and return Receipts

## Define an intent

Start by defining the schema for your payment intent. The schema specifies what parameters the server requests and what the client provides.

```ts twoslash
import { Intent, z } from 'mpay'

const charge = Intent.from({
  name: 'charge',
  schema: {
    request: z.object({
      amount: z.string(),
      currency: z.string(),
      recipient: z.string(),
    }),
  },
})
```

## Create a method intent

Extend the base Intent with method-specific request fields and Credential payload schemas. The following example uses Lightning Network:

```ts twoslash
import { Intent, MethodIntent, z } from 'mpay'

const charge = Intent.from({
  name: 'charge',
  schema: {
    request: z.object({
      amount: z.string(),
      currency: z.string(),
      recipient: z.string(),
    }),
  },
})
// ---cut---
const lightningCharge = MethodIntent.fromIntent(charge, {
  method: 'lightning',
  schema: {
    credential: {
      payload: z.object({
        preimage: z.string(),
      }),
    },
    request: {
      methodDetails: z.object({
        invoice: z.string(),
        paymentHash: z.string(),
      }),
    },
  },
})
```

## Client implementation

Extend the method intent with Credential creation logic. The `createCredential` function runs when the client gets a `402` response:

```ts twoslash
import { Credential, Intent, MethodIntent, z } from 'mpay'

const charge = Intent.from({
  name: 'charge',
  schema: {
    request: z.object({
      amount: z.string(),
      currency: z.string(),
      recipient: z.string(),
    }),
  },
})

const lightningCharge = MethodIntent.fromIntent(charge, {
  method: 'lightning',
  schema: {
    credential: {
      payload: z.object({
        preimage: z.string(),
      }),
    },
    request: {
      methodDetails: z.object({
        invoice: z.string(),
        paymentHash: z.string(),
      }),
    },
  },
})

declare function payInvoice(invoice: string): Promise<{ preimage: string }>
// ---cut---
const clientMethod = MethodIntent.toClient(lightningCharge, {
  async createCredential({ challenge }) {
    const bolt11 = challenge.request.methodDetails!.invoice;
    const result = await payInvoice(bolt11)

    return Credential.serialize({
      challenge,
      payload: {
        preimage: result.preimage,
      },
    })
  },
})
```

## Server implementation

Extend the method intent with verification logic. For Lightning Network, verify that the preimage hashes to the payment hash:

```ts twoslash
import { Intent, MethodIntent, Receipt, z } from 'mpay'

const charge = Intent.from({
  name: 'charge',
  schema: {
    request: z.object({
      amount: z.string(),
      currency: z.string(),
      recipient: z.string(),
    }),
  },
})

const lightningCharge = MethodIntent.fromIntent(charge, {
  method: 'lightning',
  schema: {
    credential: {
      payload: z.object({
        preimage: z.string(),
      }),
    },
    request: {
      methodDetails: z.object({
        invoice: z.string(),
        paymentHash: z.string(),
      }),
    },
  },
})

declare function sha256(data: Uint8Array): Uint8Array
declare function hexToBytes(hex: string): Uint8Array
declare function bytesToHex(bytes: Uint8Array): string
// ---cut---
const serverMethod = MethodIntent.toServer(lightningCharge, {
  async verify({ credential, request }) {
    const preimage = credential.payload.preimage
    const expectedHash = credential.challenge.request.methodDetails!.paymentHash

    const actualHash = bytesToHex(sha256(hexToBytes(preimage)))
    const isValid = actualHash === expectedHash

    return Receipt.from({
      method: 'lightning',
      reference: preimage,
      status: 'success',
      timestamp: new Date().toISOString(),
    })
  },
})
```

## Use in your app

### Client

Pass the client method intent to `Mpay.create`:

```ts twoslash
import { Credential, Intent, MethodIntent, z } from 'mpay'
import { Mpay } from 'mpay/client'

const charge = Intent.from({
  name: 'charge',
  schema: {
    request: z.object({
      amount: z.string(),
      currency: z.string(),
      recipient: z.string(),
    }),
  },
})

const lightningCharge = MethodIntent.fromIntent(charge, {
  method: 'lightning',
  schema: {
    credential: {
      payload: z.object({
        preimage: z.string(),
      }),
    },
    request: {
      methodDetails: z.object({
        invoice: z.string(),
        paymentHash: z.string(),
      }),
    },
  },
})

declare function payInvoice(invoice: string): Promise<{ preimage: string }>

const clientMethod = MethodIntent.toClient(lightningCharge, {
  async createCredential({ challenge }) {
    const bolt11 = challenge.request.methodDetails!.invoice;
    const result = await payInvoice(bolt11)

    return Credential.serialize({
      challenge,
      payload: {
        preimage: result.preimage,
      },
    })
  },
})
// ---cut---
const { fetch } = Mpay.create({
  polyfill: false,
  methods: [clientMethod],
})

const response = await fetch('https://api.example.com/premium')
```
