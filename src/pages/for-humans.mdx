# For Humans [Let your AI agents pay for what they need]

MPP (Machine Payments Protocol) lets your AI agents autonomously pay for services—no API keys, no billing dashboards, no manual signup. Just fund a wallet and let your agents work.

## How it works

```
┌─────────────┐     1. Request      ┌─────────────┐
│   Your AI   │ ─────────────────▶  │   Service   │
│   Agent     │                     │   (API)     │
│             │  ◀───────────────── │             │
│             │   2. 402 + price    │             │
│             │                     │             │
│             │   3. Pay + retry    │             │
│             │ ─────────────────▶  │             │
│             │                     │             │
│             │  ◀───────────────── │             │
└─────────────┘   4. Response       └─────────────┘
```

1. **Your agent requests a resource**—just a normal HTTP request
2. **Server responds with 402 Payment Required**—includes the price and payment details
3. **Agent pays and retries**—automatic with MPP SDK
4. **Server returns the resource**—with a receipt for the payment

No API keys to manage. No rate limits to worry about. Pay only for what you use.

## Get started in 2 minutes

::::steps

### Install the SDK

:::code-group

```bash [npm]
npm install mpay viem
```

```bash [pnpm]
pnpm add mpay viem
```

```bash [bun]
bun add mpay viem
```

:::

### Create a payment handler

```ts twoslash
import { privateKeyToAccount } from 'viem/accounts'
import { Mpay, tempo } from 'mpay/client'

const account = privateKeyToAccount('0xabc…123')

Mpay.create({
  methods: [tempo({ account })],
})
```

This polyfills the global `fetch` to automatically handle `402` payment challenges.

### Use fetch as normal

```ts
const response = await fetch('https://openrouter.payments.tempo.xyz/v1/chat/completions', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'anthropic/claude-3.5-sonnet',
    messages: [{ role: 'user', content: 'Hello!' }],
  }),
})
```

That's it. Payments happen automatically when services return `402`.

## Why MPP?

### For agent developers

- **No API key management**—agents pay per request, no signup needed
- **No rate limits**—pay for what you use, use what you pay for
- **Works everywhere**—any service that supports MPP
- **Transparent pricing**—see the price before you pay

### For service providers

- **Instant revenue**—get paid immediately in USDC
- **No billing infrastructure**—MPP handles payments
- **Usage-based pricing**—charge exactly what each request costs
- **Global by default**—accept payments from anywhere
