# `tempo` [Tempo payment method namespace]

The `tempo` namespace provides client-side payment methods for the Tempo blockchain.

## Methods

| Method | Description |
|--------|-------------|
| [`tempo.charge()`](/sdk/typescript/client/Method.tempo.charge) | One-time payments |
| [`tempo.stream()`](/sdk/typescript/client/Method.tempo.stream) | Low-cost high-throughput payments |

## Usage

```ts twoslash
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0x...')

const fetch = Fetch.from({
  methods: [
    // [!code focus:start]
    tempo.charge({ account }),
    // [!code focus:end]
  ],
})
```

### With multiple intents

Register both charge and stream methods on a single client.

```ts twoslash
import { Fetch, tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0x...')

const fetch = Fetch.from({
  // [!code focus:start]
  methods: [
    tempo.charge({ account }),
    tempo.stream({ account }),
  ],
  // [!code focus:end]
})
```

## Shared parameters

Both `tempo.charge()` and `tempo.stream()` accept these common parameters:

### account (optional)

- **Type:** `Account`

Account to sign transactions with. You can override this per call using the context.

```ts twoslash
import { tempo } from 'mpay/client'
import { privateKeyToAccount } from 'viem/accounts'

const method = tempo.charge({
  account: privateKeyToAccount('0x...'), // [!code focus]
})
```

### getClient (optional)

- **Type:** `(chainId: number) => Client`

Function that returns a viem client for the given chain ID.
