# Getting started [The mpay TypeScript library]

## Overview

The `mpay` TypeScript library provides a typed interface over the Machine Payments Protocol, from high-level abstractions to low-level primitives and building blocks.

## Install

:::code-group

```bash [npm]
npm install mpay
```

```bash [pnpm]
pnpm add mpay
```

```bash [bun]
bun add mpay
```

:::

## Quick start

This quick start guide shows you how to use `mpay` with the [`tempo` payment method](/payment-methods/tempo).

You can apply the same patterns to [other payment methods](/payment-methods).

- If you want to **consume a Payment-enabled service**, follow the [Client](#client) quick start.
- If you want to **protect a service** with Payment, follow the [Server](#server) quick start.

### Client

::::steps

#### Install peer dependencies

In this example, you use the `viem` library to instantiate an account.

:::code-group
```bash [npm]
npm install viem
```
```bash [pnpm]
pnpm add viem
```
```bash [bun]
bun add viem
```
:::

#### Define an account

Next, define an account that will be used to sign payments.

```ts twoslash [define-account.ts]
import { privateKeyToAccount } from 'viem/accounts'

const account = privateKeyToAccount('0xabc…123')
```

:::tip
When using Tempo, you can also use [Passkey or WebCrypto accounts](https://viem.sh/tempo/accounts).
:::

#### Create payment handler

Call `Mpay.create` at startup. This polyfills the global `fetch` to automatically handle `402` payment challenges.

```ts twoslash [create-paid-fetch.ts]
import { privateKeyToAccount } from 'viem/accounts'
import { Mpay, tempo } from 'mpay/client' // [!code hl]

const account = privateKeyToAccount('0xabc…123')

Mpay.create({ // [!code hl]
  methods: [tempo({ account })], // [!code hl]
}) // [!code hl]
```

:::tip
If you want to avoid polyfilling, use the returned `fetch` instead.

```ts 
const mpay = Mpay.create({
  polyfill: false, // [!code hl]
  methods: [tempo({ account })]
})

const response = await mpay.fetch('https://mpp.dev/api/ping/paid') // [!code hl]
```
:::

#### Request protected resources

Use `fetch`. Payment happens when a server returns `402`.

```ts [fetch-resource.ts]
const response = await fetch('https://mpp.dev/api/ping/paid')
```

::::

### Server

::::steps

#### Create an `Mpay` instance

Create an `Mpay` instance to request payment for protected resources.

```ts twoslash [create-mpay.ts]
import { Mpay, tempo } from 'mpay/server'

const mpay = Mpay.create({ methods: [tempo.charge()] })
```

:::tip
It is highly recommended to also set a `secretKey` to bind challenges to their contents, 
and allow the server to verify that incoming credentials match challenges it issued.

```ts
// .env // [!code focus]
MPAY_SECRET_KEY="random-secret-key" // [!code focus]

// config.ts // [!code focus]
const mpay = Mpay.create({
  methods: [tempo.charge()],
  secretKey: process.env.MPAY_SECRET_KEY!, // [!code focus]
})
```
:::

#### Use in request handlers

Use the `mpay` instance to request payment from the client.

The intent handler accepts a [Fetch API](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API)-compatible request object, and returns a `Response` object.

The Fetch API is compatible with most server frameworks, including:
- [Hono](https://hono.dev)
- [Deno](https://deno.com)
- [Cloudflare Workers](https://workers.dev)
- [Next.js](https://nextjs.org)
- [Bun](https://bun.sh)
- other Fetch API-compatible frameworks

```ts twoslash [charge-handler.ts]
import { Mpay, tempo } from 'mpay/server' // [!code collapse:8 collapsed]

const mpay = Mpay.create({ methods: [tempo.charge()] })

export async function handler(request: Request) { 
  const response = await mpay.charge({ 
    amount: '0.1', 
    currency: '0x20c0000000000000000000000000000000000000', 
    recipient: '0xa726a1CD723409074DF9108A2187cfA19899aCF8', 
  })(request) 

  // Payment required: send 402 response with challenge 
  if (response.status === 402) return response.challenge 

  // Payment verified: attach receipt and return resource 
  return response.withReceipt(Response.json({ data: '...' })) 
} 
```

:::tip
If your framework doesn't support the Fetch API (for example, Express or Node.js), you're likely interfacing with the [Node.js Request Listener API](https://nodejs.org/api/http.html#httpcreateserveroptions-requestlistener).

Use the `Mpay.toNodeListener` helper to transform the handler into a Node.js-compatible listener.

```ts  twoslash
import { Mpay, tempo } from 'mpay/server' // [!code collapse:8]

const mpay = Mpay.create({ methods: [tempo.charge()] })

type IncomingMessage = import('node:http').IncomingMessage
type ServerResponse = import('node:http').ServerResponse
// ---cut---
export async function handler(req: IncomingMessage, res: ServerResponse) { 
  const response = await Mpay.toNodeListener(mpay.charge({ // [!code ++]
    amount: '0.1', 
    currency: '0x20c0000000000000000000000000000000000000', 
    recipient: '0xa726a1CD723409074DF9108A2187cfA19899aCF8', 
  }))(req, res) // [!code ++]

  // Payment required: send 402 response with challenge 
  if (response.status === 402) return response.challenge 

  // Payment verified: attach receipt and return resource 
  return response.withReceipt(Response.json({ data: '...' })) 
} 
```

:::

#### Plug into your framework

Plug the handler into your framework of choice, like [Hono](https://hono.dev).

```ts [hono-server.ts]
import { Hono } from 'hono' // [!code focus]
import { Mpay, tempo } from 'mpay/server'

const mpay = Mpay.create({ methods: [tempo.charge()] })

const app = new Hono() // [!code focus]

app.get('/resource', async (c) => { // [!code focus:99]
  const r = await mpay.charge({
    amount: '0.1',
    currency: '0x20c0000000000000000000000000000000000000',
    recipient: '0xa726a1CD723409074DF9108A2187cfA19899aCF8',
  })(c.req.raw)

  if (r.status === 402) return r.challenge

  return r.withReceipt(c.json({ data: '...' }))
})
```

## Skills

Get up and running by using your coding agent (Claude Code, Cursor, Amp, and similar tools) with [Agent Skills](https://skills.sh). After you install it, your agent has context on how to properly integrate `mpay` with your chosen framework.

Use the [`skills` CLI](https://github.com/vercel-labs/skills) to install skills for your coding agent:

```bash [install-skills.sh]
# Install all mpay skills
$ npx skills install wevm/mpay

# Install a specific skill
$ npx skills install wevm/mpay --skill mpay-hono

# Install globally (available across all projects)
$ npx skills install wevm/mpay -g
```
