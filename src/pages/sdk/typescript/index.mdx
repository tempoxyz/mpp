# Getting Started

## Overview

The `mpay` TypeScript library provides an intuitive interface over the Machine Payments Protocol, from high-level abstractions to low-level primitives & building blocks.

## Skills

Get up and running quickly by using your coding agent (Claude Code, Cursor, Amp, etc.) with [Agent Skills](https://skills.sh). Once installed, your agent will have context on how to properly integrate `mpay` with your chosen framework.

Use the [`skills` CLI](https://github.com/vercel-labs/skills) to install skills for your coding agent:

```bash
# Install all mpay skills
npx skills install wevm/mpay

# Install a specific skill
npx skills install wevm/mpay --skill mpay-hono

# Install globally (available across all projects)
npx skills install wevm/mpay -g
```

## Quick Start

This quick start guide will demonstrate how to use `mpay` with the [`tempo` Payment Method](#TODO).

The same usage patterns can be applied to [other Payment Methods](#TODO).

- If you want to **consume a Payment-enabled service**, follow the [Client](#client) quick start.
- If you want to **protect a service** with Payment, follow the [Server](#server) quick start.

### Client


::::steps

#### Install Peer Dependencies

For this example, we will be using the `viem` library to instantiate a Tempo Account.

:::code-group
```bash [npm]
npm install viem
```
```bash [pnpm]
pnpm add viem
```
```bash [bun]
bun add viem
```
:::

#### Define a Tempo Account

First, we will define an Tempo Account that we will use to make payments.

For this example, we will use a simple [Secp256k1 (Private Key)](https://viem.sh/tempo/accounts/account.fromSecp256k1) Account. 
There are [other types](https://viem.sh/tempo/accounts) of Accounts you can use as well (e.g. Passkey, WebCrypto, etc).

```ts twoslash [example.ts]
import { Account } from 'viem/tempo'

const privateKey = '0xabc…123'
const account = Account.fromSecp256k1(privateKey)
```

#### Instantiate a Payment-enabled `fetch`

Next, we will instantiate a Payment-enabled `fetch` instance that will be used to automatically
handle requests that require payment. 

We will pass our Tempo `account` to the `fetch` instance, and also specify the Payment `method`(s) we want to use.

```ts twoslash [example.ts]
import { Account } from 'viem/tempo'
import { Fetch, tempo } from 'mpay/client' // [!code focus]

const privateKey = '0xabc…123'
const account = Account.fromSecp256k1(privateKey)

const fetch = Fetch.from({ // [!code focus]
  account, // [!code focus]
  methods: [ // [!code focus]
    tempo({ // [!code focus]
      chainId: 4217, // [!code focus]
      rpcUrl: 'https://rpc.tempo.xyz', // [!code focus]
    }) // [!code focus]
  ] // [!code focus]
}) // [!code focus]
```

:::tip
You can also polyfill the global `fetch` instance if you are feeling ambitious.

```ts 
Fetch.polyfill({
  account,
  methods: [
    tempo({
      chainId: 4217,
      rpcUrl: 'https://rpc.tempo.xyz',
    })
  ]
})

// Now, the global `fetch` instance will be payment-enabled.
const response = await fetch('https://mpp.dev/ping/paid')
```
:::

#### Use on Resources

Now, we can use our `fetch` instance to make requests to resources that require payment.

```ts [example.ts]
const response = await fetch('https://mpp.dev/ping/paid')
```

:::tip
We can also define accounts on a per-request basis under `context`.

```ts 
const fetch = Fetch.from({ 
  account, // [!code --] 
  methods: [ 
    tempo({ 
      chainId: 4217,
      rpcUrl: 'https://rpc.tempo.xyz', 
    }) 
  ] 
}) 

const response = await fetch('https://mpp.dev/ping/paid', {
  context: { 
    account, // [!code ++] 
  } 
})
```
:::

### Server

::::steps

#### Define an `Mpay` Instance

First, we will define an `Mpay` instance that we will use to request for payment
on our protected resources.

```ts twoslash [example.ts]
import { Mpay, tempo } from 'mpay/server'

const mpay = Mpay.create({
  method: tempo({
    chainId: 4217,
    rpcUrl: 'https://rpc.tempo.xyz',
  }),
  realm: 'mpp.dev',
})
```

#### Use on Request Handlers

Next, we will use the `mpay` instance to request payment from the client.

The intent handler accepts a [Fetch API](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API)-compatible request object, and returns a `Response` object.

The Fetch API is compatible with most server frameworks, including:
- [Hono](https://hono.dev)
- [Deno](https://deno.com)
- [Cloudflare Workers](https://workers.dev)
- [Next.js](https://nextjs.org)
- [Bun](https://bun.sh)
- and more...

```ts twoslash [example.ts]
import { Mpay, tempo } from 'mpay/server' // [!code collapse:7 collapsed]

const mpay = Mpay.create({
  method: tempo({
    chainId: 4217,
    rpcUrl: 'https://rpc.tempo.xyz',
  }),
  realm: 'mpp.dev',
})

export async function handler(request: Request) { 
  const response = await mpay.charge({ 
    request: { 
      amount: '1000000', 
      currency: '0x20c0000000000000000000000000000000000001', 
      recipient: '0xa726a1CD723409074DF9108A2187cfA19899aCF8', 
      expires: '2030-01-20T12:00:00Z', 
    }, 
  })(request) 

  // Payment required — send 402 response with challenge 
  if (response.status === 402) return response.challenge 

  // Payment verified — attach receipt and return resource 
  return response.withReceipt(Response.json({ data: '...' })) 
} 
```

:::tip
If you are using a framework that does not support the Fetch API (e.g. Express or Node.js), then it
is likely you are interfacing with the [Node.js Request Listener API](https://nodejs.org/api/http.html#httpcreateserveroptions-requestlistener).

You can use the `Mpay.toNodeListener` helper to transform the handler into a Node.js compatible listener.

```ts  twoslash
import { Mpay, tempo } from 'mpay/server' // [!code collapse:7]

const mpay = Mpay.create({
  method: tempo({
    chainId: 4217,
    rpcUrl: 'https://rpc.tempo.xyz',
  }),
  realm: 'mpp.dev',
})

export async function handler(req: IncomingMessage, res: ServerResponse) { 
  const response = await Mpay.toNodeListener(mpay.charge({ // [!code ++]
    request: { 
      amount: '1000000', 
      currency: '0x20c0000000000000000000000000000000000001', 
      recipient: '0xa726a1CD723409074DF9108A2187cfA19899aCF8', 
      expires: '2030-01-20T12:00:00Z', 
    }, 
  }))(req, res) // [!code ++]

  // Payment required — send 402 response with challenge 
  if (response.status === 402) return response.challenge 

  // Payment verified — attach receipt and return resource 
  return response.withReceipt(Response.json({ data: '...' })) 
} 
```

:::

#### Plug into Framework

Finally, we can plug our handler into our framework of choice, like [Hono](#TODO).

```ts twoslash [example.ts]
import { Hono } from 'hono' // [!code focus]
import { Mpay, tempo } from 'mpay/server'

const mpay = Mpay.create({
  method: tempo({ chainId: 4217, rpcUrl: 'https://rpc.tempo.xyz' }),
  realm: 'mpp.dev',
  secretKey: import.meta.env.MPAY_SECRET_KEY,
})

const app = new Hono() // [!code focus]

app.get('/resource', async (c) => { // [!code focus:99]
  const r = await mpay.charge({
    request: {
      amount: '1000000',
      currency: '0x20c0000000000000000000000000000000000001',
      recipient: '0xa726a1CD723409074DF9108A2187cfA19899aCF8',
    },
  })(c.req.raw)

  if (r.status === 402) return r.challenge

  return r.withReceipt(c.json({ data: '...' }))
})
```
